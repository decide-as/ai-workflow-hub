# Crossroads Explanations

Use this structure whenever presenting 2+ options for a decision.

## Template

```
## [Decision Point]
**TL;DR**: [what we're deciding] → pick [Name] because [single phrase].

**Background**: [relevant context — 1 sentence]
**Problem**: [what needs solving — 1 sentence]
**Constraints**: [what's off the table — 1 sentence, or "none"]

**[Name]** — [what it does]. Short-term: [impact]. Long-term: [impact]. Blast radius: [scope of damage if wrong].
**[Name]** — [what it does]. Short-term: [impact]. Long-term: [impact]. Blast radius: [scope of damage if wrong].

**Reversibility**: one-way door / two-way door
**Dependency drag**: [what this blocks or unlocks]
**Pick**: [Name] — [single most important reason].
**Success signal**: [one observable that confirms it worked]
```

## Constraints

- **TL;DR**: leads every block. Decision + recommendation in one line. Executives read this and skim the rest.
- **Background**: 1 sentence. What is already true about the situation.
- **Problem**: 1 sentence. What is broken, missing, or needed. Distinct from background.
- **Constraints**: what's non-negotiable before options are considered. Keeps options realistic.
- **Each option**: name + what it does + short-term + long-term + blast radius. Nothing else.
- **Blast radius**: scope of damage if the pick is wrong — distinct from reversibility (you can reverse and still cause harm).
- **Reversibility**: one-way door = needs more scrutiny. Two-way door = decide fast.
- **Dependency drag**: what this decision blocks or unlocks. Often the real reason one option wins.
- **Pick is mandatory**: always give a recommendation. Never end with "it depends."
- **Success signal**: one observable. Makes the recommendation falsifiable.
- **Max 4 options** per block. More than 4 = split the decision first.
- **No nested bullets** inside an option line.
